# Frontend Internals

## Transcript Entry Types

**NO `as any` on transcript entries. EVER.**

Discriminated union `TranscriptEntry` in `src/shared/protocol.ts`. Use type guards:

```ts
if (isUser(entry)) entry.message?.content  // typed
if (entry.type === 'assistant') (entry as TranscriptAssistantEntry).message?.usage?.speed
```

**Types:** `user`, `assistant`, `progress`, `system`, `queue-operation`,
`compacting`/`compacted` (synthetic), `last-prompt`, `pr-link`.

**Rendering augmentations:** Concentrator adds `images` and `toolUseResult` to entries.
Use `RenderableTranscriptEntry` (in `group-view.tsx`) for rendering access.

**Type discovery:** `python3 scripts/extract-transcript-types.py` mines real JSONL files.
Run after CC version bumps to catch format drift.

## Queue-Operation Entries

User interjects while Claude works -> `queue-operation` entries in JSONL.
- `enqueue` with user text -> queued user group (floats at bottom)
- `enqueue` with `<task-notification>` -> inline system notification (NOT queued)
- `dequeue`/`remove`/`popAll` clear the queued flag
- On reconnect, all queued flags cleared (historical state is stale)

## Skill Content Rendering

Skill content detected and collapsed into teal pill divider.

**Path A (Skill tool):** `toolUseResult.commandName` on `tool_result` -> next `isMeta` entry = content.\
**Path B (/slash):** `<command-message>name</command-message>` -> next `isMeta` entry = content.

Group type `'skill'` with `skillName` on `DisplayGroup`. Component: `SkillDivider`.

## Effort Level Tracking

`usage.speed`: `fast`=low, `standard`=medium, `extended`=high. Extracted from
assistant transcript entries in session-store.ts.

**Ultrathink:** Orange underline on keyword in prompt input overlay. Only on main
session input (`enableEffortKeywords` prop), NOT task editors or quick notes.
Critical: overlay must never add characters -- breaks caret alignment.

CC 2.1.94 changed default effort to high for API/Team users. Our `defaultEffort`
defaults to `'default'` (no `--effort` flag, inherits CC default).

## Service Worker

`asset-manifest.json` generated by Vite plugin. SW precaches all output files.
Each build = unique cache name. Old caches deleted on activate.

**Update:** Browser checks `sw.js` hourly. New SW -> `sw-updated` postMessage ->
cyan "New version available" banner.

`clearCacheAndReload()` in `utils.ts` -- used by update banner, nerd modal, settings.

## CodeMirror Syntax Highlighting

CM6's `syntaxHighlighting()` breaks when code-split (Vite async chunk). Facet identity
breaks across chunk boundaries.

**Workaround:** Direct `ViewPlugin` calling `highlightTree()` with Tokyo Night
`HighlightStyle`. Must also mount CSS via `EditorView.styleModule.of()`.

Files: `codemirror-setup.ts`, `file-editor.tsx`.

## Mermaid Diagrams

` ```mermaid ` blocks via `beautiful-mermaid` (synchronous SVG, NOT standard mermaid npm).
Pipeline: marked -> `<pre class="mermaid">` -> useEffect -> `renderMermaidSVG()` -> SVG.
Theme via CSS variables. Virtualizer = only render when visible.

## StopFailure Hook

CC 2.1.78+. Fires on API errors (rate limits, auth failures). Session `lastError`:
`stopReason`, `errorType`, `errorMessage`, `timestamp`. Red badge in list, red banner
in detail. Clears when session resumes.
